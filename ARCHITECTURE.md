# 🏗️ 系统架构设计文档

## 📖 文档概述

本文档详细描述了 Auto Test Agent 的系统架构设计，包括整体架构、模块设计、数据流、技术选型等。

## 🎯 系统目标

创建一个桌面客户端的自动化测试工具，实现：
- **一键安装**: 用户一行命令完成本地安装
- **远程控制**: 网页端发送指令 → 客户端执行 → 结果回传
- **实时监控**: 详细的执行日志、错误诊断、性能分析
- **美观UI**: 功能完整、用户友好的界面

## 📦 核心模块设计

### 1. 命令行工具 (CLI)

**位置**: `packages/cli/`

**职责**:
- 环境自检 (Node.js、Chrome、Playwright)
- 一键安装和配置
- 协议注册 (midscene://)
- 开机自启配置

**核心类**:
- `EnvironmentMonitor` - 环境检查和自动修复
- `install()` - 安装逻辑
- `setup()` - 配置管理

### 2. 桌面客户端 (Desktop Client)

**位置**: `packages/desktop-client/`

**职责**:
- 任务进度实时展示
- 详细日志查看
- 步骤时间线
- 错误诊断助手
- 性能分析图表

**核心组件**:
- `ProtocolHandler` - 处理 midscene:// 协议
- `WSClient` - WebSocket 客户端连接
- UI组件:
  - `TaskProgress` - 任务进度条
  - `LogViewer` - 日志查看器
  - `StepTimeline` - 步骤时间线
  - `ErrorDiagnosis` - 错误诊断
  - `PerformanceChart` - 性能图表

### 3. 网页控制台 (Web Console)

**位置**: `packages/web-console/`

**职责**:
- 任务管理 (创建、编辑、删除)
- 一键运行测试
- 实时监控任务进度
- 查看测试报告

**核心页面**:
- `TaskList` - 任务列表
- `TaskEditor` - 任务编辑器
- `TaskMonitor` - 任务监控
- `ReportView` - 报告查看

### 4. 任务执行引擎 (Task Executor)

**位置**: `packages/task-executor/`

**职责**:
- 基于用户的任务描述自动生成可执行的操作脚本
- 执行 Playwright 操作流
- 性能监控
- 错误恢复
- 截图捕获

**核心类**:
- `TaskRunner` - 任务运行器
- `MidsceneAdapter` - @midscene/web 适配器
- `AutoOptimization` - 自动优化 (网络空闲检测、智能重试)
- `PerformanceMonitor` - 性能监控
- `ErrorRecovery` - 错误恢复
- `VisualEvidence` - 视觉证据 (截图)
- `LocalLog` - 本地日志存储
- `ResultUploader` - 结果上传

### 5. 后端服务器 (Server)

**位置**: `packages/server/`

**职责**:
- 任务管理 API
- 客户端管理
- 实时通信 (WebSocket)
- 报告存储和查询

**核心模块**:
- `TaskAPI` - 任务 CRUD API
- `ClientAPI` - 客户端管理 API
- `ReportAPI` - 报告 API
- `WebSocketHandler` - WebSocket 处理器
- `TaskStore` - 任务存储
- `ClientStore` - 客户端状态存储
- `ReportStore` - 报告存储

### 6. 共享模块 (Shared)

**位置**: `packages/shared/`

**职责**:
- TypeScript 类型定义
- 常量定义
- 工具函数
- 配置管理

**核心内容**:
- `types.ts` - 核心类型定义
- `events.ts` - WebSocket 事件定义
- `config.ts` - 配置管理
- `utils.ts` - 工具函数

### 通信协议

#### HTTP 通信
- **用途**: 任务 CRUD、报告查询
- **协议**: RESTful API
- **格式**: JSON

#### WebSocket 通信
- **用途**: 实时任务进度、状态同步
- **协议**: Socket.io
- **事件**: 客户端注册、任务分配、进度更新、完成通知

#### 自定义协议
- **用途**: 唤起桌面客户端
- **协议**: `midscene://`
- **示例**: `midscene://run?taskId=xxx`

## 🔐 安全考虑

### API安全
- CORS配置：限制允许的来源
- 输入验证：所有API端点验证输入
- 错误处理：不暴露敏感信息

### WebSocket安全
- 身份验证：客户端注册机制
- 消息加密：生产环境使用WSS
- 速率限制：防止恶意连接

### 数据安全
- 本地存储：敏感数据加密
- 日志脱敏：隐藏敏感信息
- 访问控制：基于角色的权限

## ⚡ 性能优化

### 服务端
- 连接池：数据库连接复用
- 缓存策略：热点数据缓存
- 异步处理：耗时任务异步化

### 客户端
- 懒加载：组件按需加载
- 虚拟滚动：大量数据优化
- 图片优化：截图压缩

### 通信优化
- 流式传输：实时数据推送
- 批量处理：减少网络请求
- 压缩传输：数据压缩

## 🚀 扩展性设计

### 水平扩展
- 无状态服务器：支持多实例
- 负载均衡：请求分发
- 数据库分片：数据分区

### 功能扩展
- 插件系统：支持自定义插件
- 脚本扩展：支持自定义脚本
- UI主题：支持主题定制

### 集成扩展
- 第三方服务：支持外部API集成
- 消息队列：支持异步任务队列
- 云存储：支持云存储服务

## 📊 监控和运维

### 日志系统
- 结构化日志：统一的日志格式
- 日志级别：DEBUG/INFO/WARN/ERROR
- 日志轮转：自动清理旧日志

### 性能监控
- 关键指标：响应时间、吞吐量
- 资源监控：CPU、内存、磁盘
- 告警机制：异常情况通知

### 健康检查
- 服务状态：定期检查服务健康
- 依赖检查：检查外部依赖
- 自动恢复：故障自动恢复

## 🎯 设计原则

1. **模块化**: 每个模块职责单一，高内聚低耦合
2. **可扩展**: 支持水平扩展和功能扩展
3. **安全性**: 多层安全防护
4. **性能**: 优化关键路径，提升用户体验
5. **可维护**: 清晰的代码结构，完善的文档
