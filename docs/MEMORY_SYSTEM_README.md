# 📖 项目内存系统说明

## 什么是内存系统？

本项目使用 Claude Code 的持久化内存系统来存储项目上下文信息，确保在新的对话中可以快速恢复项目状态。

## 📂 内存文件位置

内存文件存储在: `C:\Users\96425\.claude\projects\G--code-auto-test-agent\memory\`

## 📋 内存文件清单

### 1. MEMORY.md (索引文件)
内存系统的主索引，列出所有可用的内存文件。

### 2. user_profile.md (用户角色档案)
记录用户的基本信息、工作偏好、技术背景和决策风格。

**包含内容**:
- 用户角色和项目目标
- 工作偏好和沟通风格
- 技术栈偏好
- 工作进展和当前重点
- 重要反馈记录

### 3. project_status.md (项目状态总结)
记录项目的当前状态、已完成功能、进行中功能和技术债务。

**包含内容**:
- 项目概况和技术栈
- 已完成功能列表
- 进行中功能列表
- 文档体系概览
- 技术债务和待解决问题
- 开发路线图

### 4. development_log.md (开发会话记录)
记录详细的开发历史、技术决策和经验总结。

**包含内容**:
- 会话信息和主要工作
- 发现的问题和解决方案
- 重要技术决策
- 成果统计和质量提升
- 经验总结和技术洞察

### 5. technical_specs.md (技术规范和约定)
记录项目的编码规范、架构原则和工作流程。

**包含内容**:
- 项目规范和代码风格
- 技术约定和最佳实践
- UI/UX设计约定
- 开发规范和测试要求
- 持续改进指南

## 🔍 如何使用内存系统

### 在新的对话中
1. Claude Code 会自动加载这些内存文件
2. 快速了解用户角色和项目状态
3. 避免重复解释项目背景
4. 保持工作连续性

### 更新内存文件
当以下情况发生时，应该更新相应的内存文件：
- 用户提供了新的反馈 → 更新 `user_profile.md`
- 项目状态发生重大变化 → 更新 `project_status.md`
- 完成了重要功能 → 更新 `development_log.md`
- 做出了技术决策 → 更新 `technical_specs.md`
