# 清空日志功能说明

## 🎯 功能描述

在网页控制台中新增了**清空日志按钮**，方便用户清理执行日志。

## 🎨 UI 布局

```
┌─────────────────────────────────────────────┐
│ 📋 Mock 任务                               │
│                                             │
│ 任务详情和步骤                               │
│                                             │
│ ┌─────────────┐  ┌─────────────────────┐   │
│ │  🚀 START   │  │  🗑️ 清空日志        │   │
│ └─────────────┘  └─────────────────────┘   │
└─────────────────────────────────────────────┘

┌─────────────────────────────────────────────┐
│ 📊 执行日志                      5 条      │
│                                             │
│ [日志显示区域...]                           │
└─────────────────────────────────────────────┘
```

## 🔧 功能特性

### 清空日志按钮
- **颜色**: 红色 (#f44336)
- **位置**: START 按钮旁边
- **图标**: 🗑️
- **文本**: "清空日志"

### 行为
1. 点击按钮清除所有日志
2. 自动添加系统消息：`🗑️ 日志已清空`
3. 执行任务时自动禁用

### 日志计数
- 在日志区域标题显示当前日志数量
- 格式：`{数量} 条日志`
- 实时更新

## 🧪 测试步骤

### 1. 访问网页控制台
```
浏览器打开: http://localhost:5173
```

### 2. 观察界面
- 左侧：任务详情和双按钮
- 右侧：执行日志和计数

### 3. 测试清空功能
```
1. 点击 START 按钮（生成一些日志）
2. 观察日志累积
3. 点击"清空日志"按钮
4. 查看日志被清空
```

### 4. 验证禁用状态
```
1. 再次点击 START 按钮
2. 在执行过程中尝试点击"清空日志"
3. 确认按钮被禁用（灰色）
4. 任务完成后恢复可用
```

## 📱 用户体验

### 正常状态
- START 按钮：绿色，可点击
- 清空日志：红色，可点击

### 执行中状态
- START 按钮：灰色，禁用，显示"⏳ 执行中..."
- 清空日志：灰色，禁用

### 清空后反馈
```
[14:23:15] 🗑️ 日志已清空
```

## 💡 使用场景

1. **清理旧日志** - 在多次测试后清理界面
2. **重新开始** - 清空之前的日志，开始新的测试
3. **界面整洁** - 保持日志区域干净，便于查看新日志
